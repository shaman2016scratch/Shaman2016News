Интерграм выпустил Intergram-QR.

Первая версия выложена прямо в ванильном Скретче без модов, она доступна по ссылке: https://scratch.mit.edu/projects/1236439150

Что есть?

• Список символов и список всех доступных кодов

• Генерация QR-кода по списку символов, списку кода и спискам цчетов и яркости

Чего из ожидаемого нет?

• Распознаватия QR-котов по изображению, потому что ванильный Скретч и многие его моды не позволяют узнать каких цветов касается спрайт

• большого размера. Размер генерируемого QR-кода составляет 21×22 пикселей, или же 444 полей, в самой же базе 26 символов и 676 уникальных кодов (одно поле в QR-коде — один код, одно поле в QR-коде = один пиксель в сгенерированом QR-коде)

Что необычного

• Пиксели в QR-коде разного цвета и разной яркости, завичит от кодов в QR-коде

Как он устроен?

Там находятся списки "Символы" (из 26 элементов), "Цвет" (из 26 элементов), "Яркость" (из 26 элементов)

Дальше там находится список "Кодировка" где находятся все 676 уникальных кодов которые только можно было составить.

Дальше, каждый код имеет длину "2", оба символа из списка "Символы", первый символ в коде значит цвет а второй яркость.

Дальше, список "Код" который используется для генерации QR-кода и состоит из 444 элементов, один его элемент один код.

В примере указаны первые 444 кодов из списка "Символы" а так список "Код" может состоять из любых элементов списка "Символы", элементы в списке "Код" могут повторяться.

Дальше, код делает рисунок (сам QR-код) из 22 строк и 21 столбца

Для создания QR-кода код задаёт переменной 'i' значение '0' и при создании каждого пикселя QR-кода он задаёт перу размер 1 пиксель, изменяет переменную 'i' на один, получает код из элемента списка "Код" номер которого равен значению переменной 'i', задаёт цвету пера значение которое равно элементу в списке "Цвет" номер которого равен элементу списка "Символы" значение которого равно первому символу в коде из элемта списке "Код" номер которого равен значению переменной "i" и задаёт яркости пера значение которое равно элементу в списке "Яркость" номер которого равен элементу списка "Символы" значение которого равно второму символу в коде из элемента списка "Код" номер которого равен значению переменной "i" и опускает перо, пиксель появляется и перо поднимается
